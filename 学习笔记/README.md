## 📋 笔记规范

### 文件命名规范
- **理论笔记**: `01-理论概念.md`
- **编程实践**: `02-编程练习.md`
- **问题总结**: `03-问题与解决.md`
- **扩展思考**: `04-扩展思考.md`
- **参考资料**: `05-参考资料.md`

### 笔记内容结构
```markdown
# [讲座名称] - 学习笔记

## 📚 核心概念
## 💻 编程实践
## ❓ 问题与解决
## 💡 深入思考
## 📖 参考资料
```

### 学习状态标识
- 🟢 **已完成**: 所有内容学习完毕，练习完成
- 🟡 **进行中**: 正在学习过程中
- 🔴 **遇到困难**: 学习中遇到需要帮助的问题
- ⚪ **未开始**: 尚未开始学习

## 💡 使用建议

1. **及时记录**: 学习过程中随时记录重要概念和问题
2. **代码实践**: 每个理论概念都要有对应的代码实现
3. **问题追踪**: 遇到的问题要详细记录和解决过程
4. **定期回顾**: 每周回顾笔记，查漏补缺
5. **知识链接**: 建立不同讲座之间的知识联系

---

**📌 提示**: 笔记是学习的重要工具，请保持记录的习惯，这将为你的学习提供宝贵的参考资料。
